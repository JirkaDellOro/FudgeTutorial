FUDGE repository

Getting started

To begin, you'll need to establish your working environment.
Here, we'll guide you through the process of installing the FUDGE repository in your environment and provide your first look at the FUDGE editor.


Create a GitHub account

A new account can be created here.

This step is optional, as you can download the repository locally without needing an account.
Keep in mind, though, that any future engine updates won't be automatically applied, and you'll need to manually download new versions.


Download/clone FUDGE-Repository

Open the FUDGE repository via:
  • Code -> Clone
  • via GitHub Desktop and clone the repository

As mentioned earlier, you can simply download the repository as a ZIP locally to your working environment.

Install FUDGE

In order to install FUDGE, you need to make sure that you have the following installed in addition to the local files:

  • download nodejs Nodejs - needed to install FUDGE and get it running.
  • TypeScript - This is optional, but recommended to write custom scripts in FUDGE. To install TypeScript globally, type 
    npm install -g typescript
    in the terminal. Alternatively, you can use JavaScript to write the scripts.

Once this is done, follow the steps below:

  • Extract the ZIP file into the desired folder.

  • Install the needed modules via 
    npm install --omit=dev
    in the folder where FUDGE is included or just run the install.bat(Windows only). If the repository has been cloned, you can skip the first step.

Start editor

to open the FUDGE editor, you have the following options:

  • type electron in the terminal 

    editor/electron

  • alternatively you can use 
    npm run-script run

  • use the FUDGE.bat file in the folder (Windows only)
